# Guitar Practice

Ритм-лаборатория (метроном + генератор ритмических паттернов) и тренажёр
грифа (формы EDCAG + боксы лада, транспонизация по всем 12 тональностям,
аккордовые прогрессии). PWA на чистых HTML/CSS/JS, без сборки и бэкенда,
работает офлайн.

Ориентация пока не залочена (раньше было только landscape под iPad на
подставке) — сейчас это осознанно временное состояние прототипа, полноценная
адаптивная вёрстка под разные платформы будет отдельной задачей.

## Экраны

- **Ритм** — метроном (BPM, TAP-темп, акцент на 1-ю долю) с переключателем
  разрешения клика 4/4 (по доле) / 4/8 (восьмыми) / 4/16 (шестнадцатыми),
  синхронизированным с единой 16-ячеечной сеткой такта, и генератором
  случайных ритмических паттернов на этой же сетке.
- **Гриф** — переключатель EDCAG/BOX поверх общего грифа (0–17 лад):
  - **EDCAG** — 5 стандартных открытых форм (E-D-C-A-G), транспонируются
    баррэ-сдвигом в любую из 12 тональностей, dur/moll меняет терцию на b3.
  - **BOX** — 7 боксов лада (3-notes-per-string), тоже транспонируются по
    тональности; под грифом в этом режиме — список мелодических секвенций
    для отработки (терцовая, квартовая и т.д.).
  - Общий для обоих режимов блок "Ступени гаммы" + аккордовые прогрессии
    (виден только в EDCAG, см. ниже) считается табличной транспонизацией
    из `theory.js`.

## Структура репозитория

```
index.html                — разметка обоих экранов
css/styles.css             — светлая тема (токены см. в reference/)
js/
  app.js                   — навигация между вкладками
  theory.js                — табличная транспонизация аккордов (формула, не хардкод)
  fretboard.js              — SVG-рендерer всего грифа (0-17 лад) для экрана "Гриф"
  fretboard-screen.js       — экран "Гриф": состояние, тональности, EDCAG/BOX, прогрессии
  rhythm-screen.js          — экран "Ритм": метроном + генератор паттернов
  data/
    caged.js                — 5 форм EDCAG (готовые аппликатуры, degree '1'/'3'/'5')
    boxes.js                 — 7 боксов лада (статические, проверены вручную на гитаре)
    progressions.js          — аккордовые прогрессии (ступенями гаммы)
    sequences.js              — мелодические секвенции для отработки в BOX-режиме
manifest.json, service-worker.js, icons/ — PWA
tests/theory.test.js        — unit-тесты транспонизации (node tests/theory.test.js)
reference/                  — исходные дизайн/логические референсы (HTML-прототипы),
                               по которым переделан текущий дизайн; не часть приложения
```

## Как считаются аппликатуры (EDCAG и боксы)

- **EDCAG** (`js/data/caged.js`) — 5 форм заданы как обычные открытые аккорды
  (C, A, G, E, D) в их естественной открытой позиции: струна/лад/ступень
  гаммы (`degree`: '1' тоника, '3' терция, '5' квинта). Это стандартная,
  общеизвестная аппликатура. Транспонизация в другую тональность (баррэ) —
  сдвиг всех ладов на разницу полутонов между выбранной тональностью и
  "родной" тоникой формы, см. `shapeAnchor()` в `fretboard-screen.js`. В
  миноре терция (`degree: '3'`) опускается на полтона (`getShapeNotes()`).

- **Боксы** (`js/data/boxes.js`) — статические данные, сверенные вручную на
  гитаре, а не вычисленные формулой. Раньше боксы генерировались функцией
  `generateBoxes()` по формуле мажорной гаммы — при ручной проверке на
  инструменте формула разошлась с реальностью, поэтому от неё отказались в
  пользу проверенных констант (`BOX_NATIVE_ANCHOR_E` — якорный лад каждого
  бокса в тональности E, `degree` — ступень гаммы '1'..'7' относительно
  этого якоря). Транспонизация — сдвиг якоря на разницу полутонов, см.
  `boxAnchor()` в `fretboard-screen.js`.

**Если понадобится добавить что-то новое:**
- Новую **форму EDCAG** — добавить объект в `shapes` в `caged.js` с реальной
  открытой аппликатурой (лад/струна/degree), транспонизация заработает
  автоматически.
- Новый **набор боксов** — проверить вручную на гитаре и внести как ещё один
  статический элемент в `boxes.js` (по образцу существующих); формулой их
  больше не выводим.

## Git-flow и деплой

- `main` — прод: GitHub Pages деплоит эту ветку при каждом коммите
  (Settings → Pages → Deploy from a branch → `main` / `/root`).
- `dev` — вся новая разработка: тестируется локально в браузере, затем через
  Pull Request `dev → main` попадает в прод.
- Тегами (`vX.Y.Z`) отмечаются состояния, соответствующие смерженным в `main`
  версиям дизайна/функциональности.

## Установка на iPad/телефон как приложение (PWA)

1. Открыть Safari (на iOS/iPadOS — обязательно Safari, не Chrome) и зайти на
   адрес GitHub Pages из шага выше.
2. Один раз зайти на сайт с интернетом — service worker скачает и закэширует
   все файлы. Дальше сайт открывается и работает офлайн.
3. Нажать кнопку **"Поделиться"** (квадрат со стрелкой вверх) в панели Safari.
4. В списке действий выбрать **"На экран «Домой»"** (Add to Home Screen).
5. Подтвердить название и нажать **"Добавить"**.
6. На домашнем экране появится иконка "Guitar Practice" — открывается как
   отдельное приложение, без адресной строки Safari, в полноэкранном режиме.
7. Service worker работает по стратегии "сначала сеть": если есть интернет, он
всегда берёт свежие файлы с GitHub Pages и обновляет офлайн-кэш; если
интернета нет — отдаёт то, что закэшировано. Поэтому после `git push` обычно
достаточно.

## Тесты

Табличная транспонизация аккордов проверяется unit-тестами на 5 известных
тональностях (C-мажор, A-минор, G-мажор, E-минор, F-мажор):

```
node tests/theory.test.js
```

Если в окружении нет Node (например, только JavaScriptCore) — та же логика
проверок продублирована в `tests/jsc_runner.js`.
