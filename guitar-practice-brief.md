# Бриф проекта: практика-компаньон для гитары (PWA)

Прочитай этот файл целиком перед тем как начинать что-либо писать. Здесь — контекст и решения, согласованные заранее в отдельном обсуждении. Если по ходу работы понадобится что-то уточнить у меня — спрашивай, не додумывай молча.

## Зачем это всё

Материалы для занятий на гитаре разбросаны: метроном — отдельный софт, CAGED-аппликатуры и последовательности аккордов приходится держать в голове или искать заново. Нужно собрать самое необходимое в одном месте на iPad (стоит на подставке/пюпитере во время игры), чтобы это работало офлайн — не хочется зависеть от интернета в моменте занятия.

## Scope текущей фазы (только это, ничего лишнего — максимально узкий MVP)

1. **Метроном** — точный, стабильный тайминг, регулировка темпа, старт/стоп.
2. **Тренажёр CAGED** — визуализация аппликатур на грифе с подсветкой тоники (отдельно обозначить мажорную и минорную тонику — разными цветами/маркерами). Точную геометрию аппликатур беру из своих скриншотов (положу в проект) — не выдумывать позиции пальцев/лады самому, только на основе присланных изображений.
3. **Тренажёр боксов и секвенций** — визуализация гитарных "боксов" (тоже по моим скриншотам, с подсветкой тоники так же, как в CAGED) + проигрывание/отображение секвенций внутри бокса. Секвенции задаю я сам — просто наборы чисел (номера пальцев/ступеней) с названием паттерна, кладу их в конфиг.
4. **Таймер занятия** — простой секундомер с нуля (старт/пауза/сброс), без разбивки на этапы урока. Считает, сколько минут я уже занимаюсь. Не обязательно сохранять между сессиями — можно сбрасываться при перезагрузке страницы.

**Явно не делаем в этой фазе** (не добавлять "заодно"): никакого трекера прогресса/чек-листов по плану, никаких ссылок на видеоуроки, никаких личных файлов/заметок, никакой синхронизации между устройствами, никакого бэкенда.

## Технические решения (уже приняты, не пересматривать без причины)

- **Тип приложения**: PWA — статический сайт (HTML/CSS/JS), устанавливается на iPad через Safari → "Добавить на экран Домой".
- **Стек**: чистые HTML/CSS/JS, без фреймворков и сборки.
- **Офлайн-режим**: service worker кэширует все статические файлы при первом заходе — дальше работает без интернета. `manifest.json` + иконки (включая `apple-touch-icon` для iOS).
- **Ориентация экрана**: только альбомная (landscape). iPad всегда стоит на подставке в этой ориентации — адаптивный дизайн под portrait/другие размеры экрана НЕ нужен, не трать на это время.
- **Метроном**: обязательно через Web Audio API с look-ahead scheduling (планирование тиков заранее через `AudioContext.currentTime`), не наивный `setInterval` — иначе будет дрифт по времени.
- **Данные аппликатур (CAGED и боксы)**: беру из скриншотов, которые положу в репозиторий (папка `reference/` или похожая). Задача — оцифровать это в JSON (лад, струна, палец, является ли тоникой и мажорная/минорная) один раз, дальше рендерить через SVG. Не изобретать альтернативные аппликатуры, использовать ровно то, что на скриншотах.
- **Данные секвенций**: JSON-конфиг с паттернами (название + массив чисел), наполняю сам.
- **Хранение**: если что-то и хранится (например последний выбранный темп метронома) — localStorage, без него тоже нормально жить в MVP.
- **Хостинг**: GitHub Pages, отдельный репозиторий (не смешивать с Telegram-проектом). Деплой — через встроенный GitHub Pages из ветки `main`, без GitHub Actions.

## Про генерацию аккордовых последовательностей в разных тональностях — как проверить, что не наврал

Не проси модель "придумывать" аккорды в каждой тональности отдельно — это как раз то место, где легко словить галлюцинацию. Правильный подход:

1. Реализуется как **детерминированная транспонизация**: для мажорной тональности есть строго фиксированная формула ступеней — I(маж) ii(мин) iii(мин) IV(маж) V(маж) vi(мин) vii°(ум.). Для натурального минора — своя фиксированная формула. Все 12 тональностей получаются не "придумыванием", а математическим сдвигом от этой таблицы.
2. Это значит: если алгоритм verified на одной-двух тональностях, которые ты и без того знаешь наизусть (например C-мажор → C-Dm-Em-F-G-Am-Hdim, A-минор → Am-Hdim-C-Dm-Em-F-G) — то и для остальных 10 тональностей он будет верен автоматически, потому что это одна и та же формула, просто сдвинутая.
3. Поэтому попроси Claude Code: (а) реализовать именно табличную транспонизацию, а не генерацию "на глаз" по каждой тональности, (б) написать unit-тесты, которые проверяют результат для 2-3 известных тебе тональностей — если тесты зелёные и совпадают с тем, что ты знаешь сам, дальше можно доверять всем остальным тональностям без ручной проверки каждой.

## Что нужно от тебя на старте

1. Предложи структуру репозитория.
2. Реализуй метроном (look-ahead scheduling, регулировка BPM, крупные кнопки старт/стоп).
3. Дождись от меня скриншотов аппликатур CAGED и боксов — предложи формат JSON для их описания (лад/струна/палец/тоника мажор-минор), покажи мне пример на одной аппликатуре, прежде чем оцифровывать все.
4. Реализуй визуализацию грифа (SVG) с подсветкой тоники.
5. Реализуй генератор аккордовых последовательностей через табличную транспонизацию (см. раздел выше) + unit-тесты на 2-3 известные тональности.
6. Реализуй экран секвенций — рендер паттернов из JSON-конфига поверх выбранного бокса.
7. Реализуй простой секундомер (старт/пауза/сброс).
8. `manifest.json` + service worker для офлайн-установки как PWA.
9. Прежде чем писать код — задай мне уточняющие вопросы, если что-то из этого брифа неполно.

## Важно

- Работай строго в рамках scope выше — никакого трекера прогресса, ссылок на видео, адаптивного дизайна под portrait. Даже если кажется, что "заодно можно" — не надо.
- Не выдумывай геометрию аппликатур/боксов сам — только по присланным скриншотам. Если скриншотов на что-то не хватает — прямо скажи, что нужно от меня, а не додумывай.
- Не генерируй аккордовые последовательности "по наитию" отдельно для каждой тональности — только через табличную транспонизацию с тестами, как описано выше.
